# Проект Buaty Vite Frontend

## Обзор проекта

Это React-приложение, построенное с использованием Vite, предназначенное как платформа для записи на услуги красоты под названием "Buaty". Приложение предоставляет современный интерфейс для пользователей, чтобы просматривать услуги, записываться на приемы, управлять профилями и общаться через функцию чата.

### Используемые ключевые технологии:
- **React 19.1.1** - Библиотека JavaScript для создания пользовательских интерфейсов
- **Vite 7.1.7** - Инструмент сборки нового поколения для фронтенда
- **Material UI (MUI)** - Библиотека компонентов для согласованного дизайна пользовательского интерфейса
- **React Router DOM** - Решение маршрутизации для навигации
- **Axios** - HTTP-клиент для взаимодействия с API
- **React Hook Form** - Управление формами и проверка
- **Socket.IO Client** - Коммуникация в реальном времени для чата
- **Date-FNS** - Библиотека манипуляций с датами
- **Emotion** - Библиотека CSS-in-JS для стилизации

### Архитектура проекта:
- **Components**: Переиспользуемые элементы пользовательского интерфейса, организованные по функциональности (аутентификация, бронирование, формы и т.д.)
- **Pages**: Компоненты уровня маршрутов, разделенные на публичные и приватные секции
- **API**: Централизованные сервисы API и аутентификации
- **Assets**: Статические ресурсы и медиафайлы
- **Styles**: Глобальные CSS и стили, специфичные для компонентов

## Сборка и запуск

### Предварительные требования:
- Node.js (рекомендуется последняя LTS-версия)
- npm или yarn пакетный менеджер

### Инструкции по установке:
1. Установите зависимости:
   ```bash
   npm install
   ```

2. Запустите сервер разработки:
   ```bash
   npm run dev
   ```
   Приложение будет доступно по адресу http://localhost:5173

3. Для сборки в продакшен:
   ```bash
   npm run build
   ```

4. Чтобы предварительно просмотреть продакшен-сборку локально:
   ```bash
   npm run preview
   ```

5. Проверьте код на ошибки:
   ```bash
   npm run lint
   ```

### Конфигурация бэкенда:
- Приложение подключается к бэкенд API по адресу `http://localhost:3001`
- Токены аутентификации хранятся в localStorage
- Запросы к API автоматически включают заголовки авторизации при наличии
- Ответы с ошибкой 401 вызывают автоматический выход из системы

## Соглашения о разработке

### Стандарты кодирования:
- Современный JavaScript ES6+ с синтаксисом JSX
- Функциональные компоненты с React Hooks
- ESLint с правилами линтинга, специфичными для React
- Последовательные соглашения об именовании (PascalCase для компонентов, camelCase для функций)
- Компоненты следуют шаблону: управление состоянием → обработчики событий → JSX

### Структура компонентов:
- Страницы являются компонентами уровня маршрута, которые составляют несколько меньших компонентов
- Компоненты организованы по функциональности в каталоге `/src/components`
- Переиспользуемые компоненты пользовательского интерфейса находятся в `/src/components/UI`
- Формы обрабатываются с помощью React Hook Form и схем валидации
- Глобальное состояние аутентификации управляется через localStorage

### Маршрутизация:
- BrowserRouter используется для маршрутизации на стороне клиента
- Публичные маршруты (доступны без аутентификации)
- Приватные маршруты (требуют аутентификации)
- Обработка 404 для несоответствующих маршрутов

### Управление состоянием:
- Локальное состояние компонента с использованием хуков useState и useEffect
- Глобальное состояние аутентификации управляется через localStorage
- Пользовательский класс AuthService обрабатывает логику аутентификации
- Заголовки авторизации автоматически добавляются к запросам API

### Стилизация:
- Глобальные стили определены в `/src/style/global.css`
- Компоненты Material UI для согласованной системы дизайна
- Emotion для динамической стилизации при необходимости
- Google Fonts (Jost) для типографики

### Интеграция API:
- Сервис API на основе Axios с перехватчиками
- Перехватчик запросов добавляет заголовки авторизации
- Перехватчик ответов обрабатывает ошибки 401
- Отдельный сервис аутентификации для операций, связанных с аутентификацией
- Обработка ошибок с понятными сообщениями для пользователя

## Ключевые особенности

### Система аутентификации:
- Потоки входа и регистрации
- Аутентификация на основе токенов
- Автоматический выход из системы при ответах 401
- Защищенные маршруты для аутентифицированных пользователей

### Система бронирования:
- Выбор услуг и планирование встреч
- Проверка форм для деталей бронирования
- Интеграция календаря для выбора даты

### Управление профилем пользователя:
- Просмотр и редактирование профиля
- Сохранение данных пользователя

### Коммуникация в реальном времени:
- Функция чата с использованием Socket.IO
- Интерфейс мгновенных сообщений

### Адаптивный дизайн:
- Мобильная версия интерфейса
- Последовательный пользовательский интерфейс на всех устройствах
- Современный дизайн с компонентами Material UI
